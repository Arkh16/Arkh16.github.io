What is the Map?

Since only 64kb of ROM & memory can be indexed at once, a map needs to be defined to show what addresses go to what space.


        0xF000    ################################
        0xE000    ########## BANKED RAM ##########     With 256 banks of 16k, there is a total of 4MB
        0xD000    ##########   (16kb)   ##########     of total accessable memory in the 16 bit address space
        0xC000    ################################
        0xB000               [reserved]
        0xA000    #### VIDEOMEMORY & settings ####    (3,840 bytes VIDEOMEMORY & 256 bytes flags/settings)
        0x9000    ################################
        0x8000    ################################
        0x7000    ################################
        0x6000    ##########            ##########
        0x5000    ##########  ROM DATA  ##########
        0x4000    ##########   (40kb)   ##########
        0x3000    ##########            ##########
        0x2000    ################################
        0x1000    ################################
        0x0000    ################################

$0000 to $9FFF is the program ROM. This has the source code/sprite info for the entire program. $A000 to $AF00 holds Video Memory, which stores text/sprite information. $AF00 to $AFFF contains settings or flags involving the videomem. $C000 to $FFFF is the system RAM (memory). Memory can be swapped between 256 memory banks. Each contains 16k of memory. Thus, in the 16 bit (65,536 byte) address space, you can index 4MB of memory and 48kb of ROM.
